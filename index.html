How To Install Phix

Phix Is Written For PHP 5.3

Phix takes advantage of new features in PHP 5.3 to keep code quality up, and code size down.

Technical Requirements

To install phix, you will need an operating system with PHP 5.3 or later installed.

phix is a command-line (CLI) tool, which you run from a Terminal or CMD window. You don't need a web browser or a web server to use phix.

Via PEAR

The best way to install Phix is via the standard PHP pear command.

Installing On Debian Or Ubuntu

Install phix on either Debian or Ubuntu using the PEAR installer:

# become root
sudo su -

# binary modules that may be missing
apt-get install php5-xdebug
apt-get install php5-imagick
apt-get install php5-xsl

# make sure we have the latest PEAR installer
pear upgrade pear/pear

# build the ncurses PHP extension
apt-get install libncursesw5-dev
pecl install ncurses
echo "extension=ncurses.so" > /etc/php5/cli/conf.d/ncurses.ini

# install phix
pear channel-discover pear.phix-project.org
pear -D auto_discover=1 install -Ba phix/phix4componentdev

Installing On Fedora, CentOS, or RedHat Enterprise Linux

Install phix on Fedora, CentOS, RedHat Enterprise Linux or other RPM-based Linux distros using the PEAR installer:

# binary modules that might be missing
sudo yum install php-process
sudo yum install php-pecl-imagick
sudo yum install php-pecl-xdebug

# make sure we have the latest PEAR installer
sudo pear upgrade pear/pear

# install phix
sudo pear channel-discover pear.phix-project.org
sudo pear -D auto_discover=1 install -Ba phix/phix4componentdev

Installing On Other Linux Distros

Because phix uses the standard PEAR installer, it should be very easy to install phix on other Linux distros. The only difficulty might be making sure that you have the right PHP extensions installed and enabled.

The extensions you need are:

  • XDebug, used by PHPUnit for generating code coverage metrics,
  • ImageMagick extension, used by several of the QA tools for drawing graphs and tables of metrics,
  • NCurses extension, used by phix to cope when you resize your terminal window,
  • POSIX extension, used by phix to work out when you're piping the phix's output into another process (so we can turn the colour off),
  • XSL extension, used for generating documentation (this is a work in progress)

Installing on OSX

phix should work on OSX Snow Leopard or Lion, but I haven't tried it myself yet. Please let me know how you get on with this.

Installing On Windows

I have not tried installing phix on Windows myself yet. I would expect the hard part is getting PHP 5.3 + the required PHP extensions installed. Please let me know how you get on with this.

What Gets Installed

This installs the phix command itself, plus all the QA tools that phix automatically supports for generating code review information, running unit tests, and generating documentation.

Create Your Own Extensions For Phix

Phix Was Built To Be Extensible

At the heart of Phix is a generic CLI tool, which you can add your own commands to.

Looking At The Structure Of Phix

When you install phix/phix4componentdev, this is a meta-package that asks the PEAR installer to install:

  • phix, which is a generic command-line tool, and
  • ComponentManager, which adds a set of commands to phix for creating and maintaining PHP components

Each of the phix commands is implemented in its own PHP class:

  • that lives inside the PhixCommands namespace,
  • that implements the Phix_Project\PhixExtensions\CommandInterface, and
  • that extends Phix_Project\PhixExtensions\CommandBase.

phix was built to be extensible. phix4componentdev is just an extension to phix that provides extra commands for creating and maintaining PHP components. It is very easy to add your own commands for phix.

Contribute To Phix

Phix Is For The Community

All of the source code for Phix is available on GitHub.

Checking The Code Out From GitHub

We use GitHub to publish all of our source code. phix itself is made from a collection of PHP components:

Phix And Generic Components

Component Purpose
Autoloader PSR0-compliant autoloader
CommandLineLib Parser for command-line switches and arguments
ConsoleDisplayLib Support for writing (possibly coloured) output to a CLI terminal.
ExceptionsLib Re-usable set of exceptions, and support for turning legacy PHP errors into exceptions.
phix The phix tool itself.
ValidationLib A set of re-usable validation tests, inspired by Zend Framework's support for validations.

Components For Creating And Maintaining PHP Components

Component Purpose
ComponentManager Meta-package for installing the commands for each type of PHP component.
ComponentManagerPhpLibrary The phix php-library:* commands.
ComponentManagerShared Code shared by all of the ComponentManagerPhp* components.
phix4componentdev Meta-package that installs all the other components, plus all the third-party tools to help with QA, code reviews, and publishing your own PEAR channel.
TasksLib A set of reusable actions (such as creating directories, copying files, etc etc) with standardised error handling.
